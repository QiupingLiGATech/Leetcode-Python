# 5. Longest Palindromic Substring

Given a string s, return the longest palindromic substring in s.

## Thoughts and Approaches

Generally, we iterate through $i = 0, 1, 2, \dots, n-1$ as the center of an odd palindrome and expand to both sides:

* Initialize l = r = i.
* If s[l] = s[r], expand to both the left and right sides by decrementing l by one and incrementing r by one, then continue checking if the longer substring is a palindrome. Do this until the indices go out of bounds or s[l]!=s[r]
* When the loop ends, check: If its length (r - l - 1) is greater than the length of our current answer, update the left and right boundaries of the answer to l+1 and r-1 so we can easily output the exact substring later.

Similarly, iterate through i and i+1 as the centers of an even palindrome—meaning we initialize l = i and r = i+1 and follow the exact same process as above.

## Python Code

```Python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n=len(s)
        SubStringL=0
        SubStringR=0

        ## When longest Palingdrome is odd length : expand from center 
        
        for i in range(n): ## find the center, so I can expand later 
            l=r=i ## i=2

            while l>=0 and r<=n-1 and s[l]==s[r]: #首先鉴定i是不是center？然后l,r 在i的基础上扩展. 
                l-=1 ## l=0
                r+=1 ## r=4
            
            if r-l-1>SubStringR-SubStringL:            
                SubStringL=l+1
                SubStringR=r  ## 左闭右开区间
                
        
        ## When longest palindrome is even length: 
        
        for i in range(n-1): ## note: i maximum is (n-2), because r=i+1
            l=i ## i=1
            r=i+1
            
            while l>=0 and r<=n-1 and s[l]==s[r]:
                l-=1 #l==0
                r+=1 ## r==3
            
            if r-l-1>SubStringR-SubStringL:            
                SubStringL=l+1
                SubStringR=r  ## 左闭右开区间
                       
        return s[SubStringL:SubStringR]
```

## Time Complexity
O(n^2) : There are 2n−1=O(n) centers. For each center, we call expand, which costs up to O(n). Although the time complexity is the same as in the DP approach, the average/practical runtime of the algorithm is much faster. This is because most centers will not produce long palindromes, so most of the O(n) calls to expand will cost far less than n iterations.
The worst case scenario is when every character in the string is the same.
## Space Complexity
O(1) : We don't use any extra space other than a few integers. This is a big improvement on the DP approach.
